# 產品需求文件 (PRD)

## 專案名稱
線上桌遊平台 (Online Board Game System)

## 專案描述
提供一個讓玩家可以隨時隨地與好友或隨機玩家進行線上桌遊對戰的網頁互動平台，解決線下難以湊齊人數或受限於實體場地的問題。

---

## 1. 專案概述

### 背景與動機
現代人生活忙碌，想要體驗桌遊的樂趣時，常遇到「找不到伴」、「沒有合適場地」或「購買實體擴充版本花費高昂」等痛點。為此，我們希望打造一個線上桌遊平台，讓熱愛桌遊的玩家能突破時間與地理限制，隨時開啟一場休閒或心機策略的遊戲。

### 目標用戶
- 喜歡玩桌遊但難以在線下湊齊朋友的玩家。
- 想要透過遊戲結交新朋友的遊戲愛好者。
- 想在空閒時間（如午休、通勤）快速進行一局體驗的學生或上班族。

### 核心價值主張
- **便利性**：免下載應用程式，透過網頁瀏覽器即可與朋友連線遊玩。
- **社交互動**：內建聊天系統與房間機制，遊戲與交友兩不誤。
- **快速組局**：簡易的建房與加入機制，迅速開始一場刺激的遊戲。

---

## 2. 功能需求

以下列出平台的核心功能，並以使用者故事（User Story）格式描述：

1. **會員帳號與登入系統**
   - 作為 **訪客**，我希望 **能夠註冊並建立帳號**，以便 **記錄我的遊戲勝率與戰績**。
   - 作為 **註冊玩家**，我希望 **能夠安全地登入系統**，以便 **存取個人資料並參與遊戲**。

2. **遊戲大廳與房間管理**
   - 作為 **玩家**，我希望 **能夠瀏覽大廳中所有進行中或等待中的房間清單**，以便 **選擇感興趣的房間加入**。
   - 作為 **房主**，我希望 **能夠建立新房間並設定遊戲人數及密碼**，以便 **能夠選擇與誰一起遊玩（公開開局或好友私局）**。

3. **遊戲核心進行與狀態同步**
   - 作為 **玩家**，我希望 **遊戲過程中的操作（例如：出牌、投票、選擇動作）能即時同步給所有房間內的玩家**，以便 **確保遊戲順暢進行並維持公平性**。
   - 作為 **系統**，我希望 **能夠根據桌遊規則判定勝負條件**，以便 **在遊戲結束時自動結算並發放積分**。

4. **即時互動與聊天系統**
   - 作為 **玩家**，我希望 **能在房間內使用文字傳送訊息**，以便 **與其他玩家進行遊戲策略討論與日常聊天**。

5. **戰績排行榜與個人中心**
   - 作為 **玩家**，我希望 **能查看全服的積分排行榜與個人歷史戰績**，以便 **了解自己的實力水平並獲得成就感**。

## 3. 非功能需求

### 技術限制
- **後端框架**：Flask
- **模板引擎**：Jinja2
- **資料庫**：SQLite (適用於輕量級資料儲存與開發)
- **前端技術**：HTML, CSS, JavaScript (Vanilla)

### 效能與安全考量
- **即時性機制**：線上桌遊強調互動的即時性，需評估採用長輪詢 (Long Polling)、AJAX 定期更新或搭配 WebSocket (如 Flask-SocketIO) 來減少狀態同步延遲。
- **資訊安全**：
  - 密碼等機密資料必須進行安全加密（Hash）後才存入資料庫。
  - 避免注入攻擊：所有資料庫存取需使用參數化查詢 (Parameterized Queires) 或 ORM，前端渲染利用 Jinja2 自動轉義防範 XSS。
- **狀態防偽**：所有的遊戲邏輯需以後端判定為準，避免玩家竄改前端資料作弊。
